Fast sRGB Gamma Approximation - Optimization Notes
==================================================

The Goal
--------
To quickly approximate the sRGB gamma correction curve: powf(x, 1/2.2) ≈ x^0.4545
Standard polynomials (x, x^2, x^3) struggle with fractional roots, so we fit a polynomial using fractional basis functions.

The "Free" Basis Function Insight
---------------------------------
In our old 3-term polynomial, we computed:
- s = x^0.5   (via sqrtf)
- q = x^0.25  (via sqrtf(s))
- x = x^1     (original input)

The breakthrough: Since `s` and `q` are already computed and sitting in CPU registers, we can multiply them together to get a brand new basis function for essentially zero cost (1 extra float multiplication):
- sq = s * q = x^0.75

Why the 4-Term Polynomial is Better
-----------------------------------
The new formula becomes:
f(x) = a*q + b*s + c*sq + d*x

Adding x^0.75 gives the curve-fitting algorithm more flexibility to match the target x^0.4545 curve:
- The x^0.25 term provides the sharp lift near 0 (shadows).
- The x^0.5 and x^0.75 terms smooth out and shape the midtones.
- The x^1 term corrects the extreme tail so that pure white maps perfectly to f(1) = 1.

Minimax Optimization Results
----------------------------
Using a script to minimize the *maximum relative error* (Minimax optimization), we found the optimal coefficients:
a = 0.04400, b = 1.17400, c = -0.28500, d = 0.06700

By adding this single "free" `sq` term, the maximum error across the entire color range dropped from ~1.7% down to just ~0.3%.

Conclusion
----------
We achieved a ~5.6x increase in mathematical precision for the microscopic cost of a single CPU multiplication cycle. A perfect example of squeezing maximum accuracy out of variables the CPU is already processing.
